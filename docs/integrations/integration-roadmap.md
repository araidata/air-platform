# Integration Roadmap

Integrations should follow operational maturity. They are not part of the first usable platform.

## Phase A - Integration Readiness

- Add integration mapping tables.
- Add outbound event model.
- Add integration status UI.
- Define retry behavior.
- Define evidence export policy.

## Phase B - Notifications

- Email notifications.
- Microsoft Teams notifications.
- Owner assignment notifications.
- AIRB review reminders.
- SLA overdue reminders.

## Phase C - Work Management

- Jira issue creation from findings.
- ServiceNow task creation from findings.
- Status synchronization.
- External ticket links in finding detail.

## Phase D - Governance Platform

- OneTrust assessment mapping.
- OneTrust risk acceptance mapping.
- OneTrust evidence metadata sync.
- OneTrust exception workflow integration.

## Phase E - Model Provider Execution

- Azure OpenAI controlled test execution.
- OpenAI API controlled test execution.
- Anthropic API controlled test execution.
- Evidence-preserving prompt and output capture.

## Integration Principle

External systems extend workflow reach. They do not replace the platform's core assurance record.
