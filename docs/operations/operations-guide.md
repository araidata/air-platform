# Operations Guide

## Daily Operations

- Review executive dashboard.
- Review critical and high findings.
- Review blocked deployments.
- Review AIRB queue.
- Review overdue SLAs.
- Check evidence completeness.
- Update remediation and retest status.

## Weekly Operations

- Review high-risk systems.
- Review findings by department.
- Review accepted risks nearing expiration.
- Review systems with stale evidence.
- Prepare AIRB review packet.
- Generate governance reports.

## Monthly Operations

- Review score trends.
- Review policy control mappings.
- Review scanner coverage.
- Review integration backlog.
- Test backup restore.
- Review user access.

## Incident-Like Events

Treat these as elevated operational events:

- Critical prompt injection finding on public-facing system.
- Possible data leakage.
- Bias/civil-rights finding in public benefits, housing, law enforcement, emergency services, or HR screening.
- Deployment attempted while blocked.
- Missing audit evidence for high-risk system.

## Runbook Expectations

Future runbooks should cover:

- Restore database.
- Restore evidence volume.
- Re-run seed data in development.
- Execute scanner adapter.
- Export governance report.
- Block deployment.
- Approve with exception.
