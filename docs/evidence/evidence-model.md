# Evidence Model

Evidence is the basis for auditability, scoring, review decisions, remediation, and exports.

## Evidence Types

- `raw_log`
- `screenshot`
- `uploaded_file`
- `scanner_output`
- `prompt`
- `model_response`
- `export`
- `note`
- `reference`

Earlier conceptual evidence types map into these Phase 2 records:

- `raw_scanner_output` and `scanner_log` -> `scanner_output` or `raw_log`
- `prompt_output` -> paired `prompt` and `model_response`
- `uploaded_report` -> `uploaded_file`
- `assessment_note` -> `note`
- `screenshot`
- `dataset_summary`, `policy_document`, `review_decision`, `risk_acceptance`, and `retest_result` -> `reference`, `note`, `export`, or `uploaded_file` depending on storage form

The `evidence` table currently stores:

- `id`
- `finding_id`
- `assessment_id`
- `system_id`
- `evidence_type`
- `title`
- `description`
- `file_path`
- `raw_text`
- `content_type`
- `created_by`
- `created_at`
- `hash`
- `metadata_json`

Evidence creation is handled through `EvidenceService`, which validates links, derives assessment/system references from a finding when possible, and creates an `evidence_created` audit event.

## Evidence Principles

- Preserve raw material.
- Keep evidence immutable when practical.
- Use new records for updates rather than overwriting history.
- Redact secrets and sensitive data where needed.
- Keep evidence linked to findings and decisions.

## Storage Strategy

Start with local or mounted storage on the single VM:

```text
evidence/
  systems/
  assessments/
  findings/
  scanner-runs/
  audit-packets/
```

Database records should store metadata and references, not necessarily large raw files.

## Chain Of Custody

For audit usefulness, capture:

- Who created or imported the evidence.
- When it was captured.
- Which run, assessment, or review created it.
- Whether it was generated by a scanner, human reviewer, or export job.
- Hash or checksum when practical.

## Sensitive Data

Evidence may contain prompts, outputs, logs, or uploaded artifacts with sensitive information. Treat evidence storage as sensitive by default.
