# Future Roadmap

The roadmap should remain governance-led. Add technical capability only when it improves assurance outcomes, auditability, or operator throughput.

## Near-Term

- Maintain the Docker Compose runtime for web, API, and PostgreSQL.
- Wire API-backed data into frontend workflows intentionally.
- Continue maturing AIRB workflow details.
- Mature Phase 6 bias and civil-rights assessment workflows.
- Add language access, human appeal, and rights-impacting review templates.

## Mid-Term

- Add additional scanner integrations only when a concrete workflow needs them.
- Add report exports.
- Add deployment approval records.
- Add Teams or email notifications.
- Add structured risk acceptance workflow.
- Add basic role-based access control.

## Later

- Add OneTrust workflow synchronization.
- Add Jira or ServiceNow ticket synchronization.
- Add scheduled scan plans.
- Add Langfuse or OpenTelemetry ingestion for selected AI systems.
- Add MLflow references for model governance where applicable.
- Add object storage for evidence.
- Add stronger authentication and authorization.

## Explicitly Not On The Early Roadmap

- Multi-tenant commercial SaaS architecture.
- Kubernetes.
- Large-scale stream processing.
- Autonomous AI agents making approval decisions.
- Reimplementation of scanner engines.
- Building a chatbot as the primary product surface.
