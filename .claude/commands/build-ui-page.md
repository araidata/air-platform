# Build UI Page

Before building a UI page, read:

- `docs/ai-context/ui-guidelines.md`
- `docs/ui-ux/design-system.md`
- `docs/ui-ux/dashboard-guidance.md`
- `docs/ui-ux/page-map.md`

Build the page as an operational dashboard surface, not a marketing page and not a chatbot.

Use:

- Calm dark professional styling.
- Readable density.
- Clear cards for repeated items.
- Findings tables.
- Evidence references.
- Risk indicators.
- Analyst drill-downs.

Avoid:

- Neon AI styling.
- Toy dashboards.
- Vague AI magic copy.
- Oversized marketing hero sections.

Use mock data unless the current phase explicitly includes backend persistence.
